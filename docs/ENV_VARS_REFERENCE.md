## EMAIL OUTREACH AUTOMATION (added 2026-05-18, schema 014)

The outreach feature lets you (and eventually each client) send AI-drafted
emails from a mailbox you own — no third-party email-vendor subscription.
Three drivers are supported and each has its own credentials.

### HostGator SMTP driver

NO env vars needed. SMTP host/user/password are entered through the UI on
`/admin/av/outreach/mailboxes` and stored encrypted in the database using
the existing `EMAIL_ENCRYPTION_KEY`.

You DO need DNS records on the sending domain for inbox placement:
- **SPF**: TXT at root, e.g. `v=spf1 +a +mx include:hostgator.com ~all`
- **DKIM**: enable in cPanel > Email Deliverability; HostGator generates the record
- **DMARC**: TXT at `_dmarc.atlanticandvine.com`, e.g. `v=DMARC1; p=quarantine; rua=mailto:dmarc@atlanticandvine.com`

### Microsoft Graph (Outlook / Microsoft 365) driver

Register an Azure app once. Then set these in Netlify:

| Variable | Purpose |
| --- | --- |
| MICROSOFT_OAUTH_CLIENT_ID | Application (client) ID from the Azure App Registration |
| MICROSOFT_OAUTH_CLIENT_SECRET | Client secret value (not the secret id) |
| MICROSOFT_OAUTH_REDIRECT_URI | `https://atlantic-hub.netlify.app/api/admin/av/outreach/mailboxes/oauth/microsoft/callback` |

Azure App Registration steps:
1. portal.azure.com → Azure Active Directory → App registrations → New registration
2. Name: "Atlantic Hub Outreach"; supported account types: "Accounts in any organizational directory and personal Microsoft accounts"
3. Redirect URI: Web → `https://atlantic-hub.netlify.app/api/admin/av/outreach/mailboxes/oauth/microsoft/callback`
4. After creation: Certificates & secrets → New client secret → copy the VALUE (not the id)
5. API permissions → Add permission → Microsoft Graph → Delegated → enable: `offline_access`, `Mail.Send`, `Mail.Read`, `User.Read`
6. Grant admin consent (if you have an org tenant)
7. Copy the Application (client) ID from the Overview tab

### Gmail / Google Workspace driver

Create a Google Cloud OAuth client once. Then set these in Netlify:

| Variable | Purpose |
| --- | --- |
| GOOGLE_OAUTH_CLIENT_ID | OAuth 2.0 client ID from Google Cloud Console |
| GOOGLE_OAUTH_CLIENT_SECRET | Matching client secret |
| GOOGLE_OAUTH_REDIRECT_URI | `https://atlantic-hub.netlify.app/api/admin/av/outreach/mailboxes/oauth/google/callback` |

Google Cloud Console steps:
1. console.cloud.google.com → new project "Atlantic Hub Outreach"
2. APIs & Services → Library → enable: "Gmail API"
3. OAuth consent screen → External → fill in app name + support email + add the three scopes: `gmail.send`, `gmail.readonly`, `userinfo.email`
4. Credentials → Create credentials → OAuth client ID → Web application
5. Authorized redirect URI: the GOOGLE_OAUTH_REDIRECT_URI value above
6. Copy the Client ID + Client Secret

## CLAY ENRICHMENT WEBHOOK (added 2026-05-20, schema 012)

The Clay receiver at `POST /api/admin/av/integrations/clay-webhook` accepts
enriched rows from Clay tables and either inserts new leads or fills missing
fields on existing leads (cross-source deduped). One shared-secret env var:

| Variable | Purpose |
| --- | --- |
| CLAY_WEBHOOK_SECRET | Shared secret. Sent by Clay in the `X-Webhook-Secret` request header. The route returns 401 if the header is missing, the secret is unset, or the value does not match (constant-time compare). |

How to generate and wire:

1. Locally: `openssl rand -hex 32` -> copy the 64-character hex string.
2. Netlify -> Site -> Environment variables -> add `CLAY_WEBHOOK_SECRET` =
   that string. Trigger a redeploy.
3. In Clay: open the table -> Webhook destination ->
   - URL: `https://atlantic-hub.netlify.app/api/admin/av/integrations/clay-webhook`
   - Add header: `X-Webhook-Secret` = the SAME value you set in Netlify.
4. Verify on `/admin/av/integrations/clay` -- the status banner reads
   "CLAY_WEBHOOK_SECRET is set" when wired correctly, and the first row
   from Clay appears in the "Last 50" table.

If the variable is unset the receiver fails closed (401) -- no rows leak in
without the secret.

## CLIENT PORTAL (added 2026-05-17c)

Both optional. Sensible defaults are baked in, so the portal works without
either of them being set; you only need them if you want to override.

| Variable | Purpose | Default |
| --- | --- | --- |
| MAGIC_LINK_BASE_URL | Origin used in the magic-link URLs we log to Netlify function output. Set this to the public origin of atlantic-hub so the link Val pastes to clients is fully qualified. | "" (empty = relative path) |
| PORTAL_ALLOWED_ORIGINS | Comma-separated origin allowlist for the cross-origin `/api/client/intake` endpoint. | https://atlanticandvine.netlify.app,https://atlanticandvine.com,https://www.atlanticandvine.com |

Recommended Netlify values:
- MAGIC_LINK_BASE_URL = https://atlantic-hub.netlify.app
- PORTAL_ALLOWED_ORIGINS = (leave unset to use the default allowlist)

No new secrets. The portal reuses the existing JWT_SECRET and JWT_ISSUER
for its session cookies.

## HOW TO ADD A VARIABLE TO NETLIFY

1. Go to netlify.com and sign in
2. Click the atlantic-hub site
3. Site configuration > Environment variables
4. Click "Add a variable" > "Add a single variable"
5. Key: the variable name (case-sensitive, all caps)
6. Value: paste the secret
7. Scopes: leave at "All scopes" (production, deploy previews, branch deploys)
8. Save
9. Netlify rebuilds automatically OR click "Trigger deploy" > "Clear cache and deploy site"

---

## HOW TO TEST A VARIABLE IS SET CORRECTLY

After deploy, hit one of the API endpoints that uses it. If the variable
is missing or wrong, my code returns a 503 with a clear error like:
- "GOOGLE_PLACES_API_KEY not configured in Netlify env vars"
- "APIFY_API_TOKEN is not set in Netlify environment variables"
- "OPENAI_API_KEY not configured in Netlify env vars"

So you don't need to guess. If a feature breaks, the error tells you which
variable is missing.

## HOSTGATOR-SIDE VARIABLES (different system)

The PHP backend at api.atlanticandvine.com uses HostGator's own environment.
Those are set in cPanel under "Software > MultiPHP INI Editor" or in the
PHP files directly. NOT in Netlify. The two systems don't share env vars.

If you need a list of HostGator-side variables, look at the .php files in
AV_livewebsite/api/ and grep for getenv() and $_ENV.
